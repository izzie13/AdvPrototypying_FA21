Final Project : Machine Learning and Gesture Detection

Version 0.0.0

The initial concept for this project is to use machine learning in order to detect anime fans' hand gestures while they are taking photos during the Seichi Junrei. ( which is the act of visiting places that were used as the stage for films, anime, cartoons, manga, etc. )

The prototype is going to provide them an immersive experience by taking photos with their favourite anime characters when they are performing the same hand gesture as the character does.

Although I am not a big fan of anime, I respect all kinds of sub-culture, and I have big interest and desire to know and help them. As a designer, I think it is my responsibility to make every main-stream community feel that their voice are been heard and taken seriously and feel included.

Background

Interact with favorite anime characters and take pictures with them is every otakus' dream, and they usually using photoshop and AR camera to help them accomplished their dreams.

Components
Version 0.0.1

In the version 0.0.1, I started to experiment with the hand gesture detection by clarify the relative position of each fingers, I can detect the victory and I love you gesture in this stage, but the detection result would influenced by the hands' pointing direction easily. To solve this problem and make the result more accurate, I need to spend time on teaching machine to learn each gesture so that the machine would identify each gesture precisely. However, due to the time constraint, I have to shift my concept into other directions.

Version 0.0.2

In the version 0.0.2, I changed my concept from detecting specific hand pose to simply using the finger movement to do something and gamify it. I started to experiment with using the index finger to draw the line. But soon, I encountered a problem of drawing separate lines instead of drawing a continuous line which is hard for user to edit their drawings.

Version 0.0.3

In this version, I figured out how to draw separate lines by reset the arrays, and I also tried to generate different shape randomly, so that user would try to use their finger to trace the shape.

Version 0.0.4

In the final version, I combined all my demos together, and designed this shape tracing game. As you can see, the pointer cannot detected precisely all the time, and I still need more time to let the machine learn how to identify each fingers.

Version 0.0.5

To finish this prototype up, I still need to work on evaluating the final drawings and give user the result. The ways I am thinking of comparing the drawing and the shape is to calculate the proportion of the overlapping points or calculate the average distance of the drawing to the shape.

Future Iteration

Although the final output is really different from my initial concept, but I learned a lot from the whole developing process, and I become more familiar with the machine learning and hand pose detection. For the future step, I still want to shift this project back to helping otakus create immersive Seichi Junrei experience, and make this game more anime-like and more interactive.
